# 若木云卡 AI 开发规则

本项目是基于 Dujiao-Next 二开的虚拟卡 / 卡密 / 数字商品交付平台。

## 项目结构

- apps/api：后端主服务；
- apps/user：客户前台；
- apps/admin：管理后台，后续供应商后台优先在这里扩展；
- docs：项目文档；
- deploy：部署配置；
- scripts：脚本。

## 总体规则

1. apps/api 是系统核心，订单、支付、交付、供应商权限、结算逻辑必须以后端为准。
2. apps/user 是客户前台，默认不得展示供应商信息。
3. apps/admin 是管理后台，后续供应商后台能力优先在 admin 中扩展。
4. 不要把 api、user、admin 混成一个应用。
5. 修改跨端功能时，必须同时检查 api、admin、user 是否需要同步调整。
6. 所有新增数据库字段必须同步记录到 docs/04-数据库变更记录.md。
7. 所有新增 API 必须同步记录到 docs/05-API接口规范.md。
8. 所有订单状态必须由后端统一定义，前端不得自行发明状态。
9. 客户不能看到供应商联系方式、供应商账号、供应商结算信息。
10. 供应商只能看到自己绑定商品产生的订单。
11. 平台管理员可以查看全部商品、订单、供应商、结算和售后信息。
12. 供应商提交的卡密、兑换码、账号信息、人工交付内容必须有操作记录。
13. 涉及支付、订单、发货、结算的逻辑必须谨慎修改，不允许无依据删除原有校验。
14. 开发前必须先阅读 README.md、AI_RULES.md 和 docs/00-项目总纲.md。
15. 每次完成一个功能后，需要说明修改了哪些端、哪些文件、是否涉及数据库变更、是否涉及 API 变更。

## 供应商隔离原则

供应商后台只能访问：

- 自己的商品；
- 自己商品产生的订单；
- 自己提交的发货记录；
- 自己的结算记录。

供应商后台不能访问：

- 其他供应商商品；
- 其他供应商订单；
- 客户完整隐私信息，除非履约必须；
- 平台总收入；
- 平台利润；
- 其他供应商结算信息。

## 客户隔离原则

客户只能看到：

- 商品信息；
- 自己的订单；
- 自己的支付状态；
- 自己的交付内容；
- 平台客服 / 售后入口。

客户不能看到：

- 供应商姓名；
- 供应商联系方式；
- 供应商后台账号；
- 供应商结算信息；
- 平台内部成本价。

## 后续重点模块

后续二开重点包括：

1. 供应商管理；
2. 商品绑定供应商；
3. 订单分配；
4. 供应商发货；
5. 平台审核交付；
6. 售后工单；
7. 供应商结算；
8. 风控审核；
9. 财务统计；
10. 平台运营看板。
